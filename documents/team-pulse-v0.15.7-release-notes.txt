Team Pulse v0.15.7 release notes

Summary
- Scanner hygiene release. No behavior changes, no schema changes, no UI changes for day-to-day use beyond one removed Settings button.
- Removed the built-in ZIP archive builder from the HTML. The "Download full archive" button and its event wiring are gone, and the six functions that implemented the ZIP writer have been deleted.
- Removed additional dead code uncovered during the sweep: seven top-level functions that were orphans (either left stranded by the ZIP removal or already unused in v0.15.6).
- Reduced innerHTML usage in the simpler render paths. Six innerHTML sites have been rewritten to textContent and createElement. Six complex-markup sites remain and are now annotated in the source with the reason they were kept.
- Compressed the CSS block inside the HTML by stripping redundant whitespace. The visual design is unchanged.

What changed

1. ZIP archive builder removed
- The `Download full archive` button has been removed from the Data health section of Settings.
- The document-level click listener no longer listens for `#downloadArchiveBtn`.
- Six functions have been deleted in full: `crc32`, `dosDateTime`, `uint16LE`, `uint32LE`, `createZipBlob`, `buildArchiveFiles`, `downloadFullArchive`. `uint16LE` and `uint32LE` were private helpers with no other callers and went with the rest.
- JSON export (`Export now`), rolling backups (`.backup.json`, `.daily.json`, `.monthly.json`), autosave, and the File System Access durability path are unchanged. Durability still works end to end, it just no longer produces a ZIP alongside the plain files.

2. Dead code removed
- While removing the ZIP builder a scanner was run over the full JavaScript block to find top-level functions and UPPER_CASE constants with no live references anywhere in the file. Seven orphan functions were found and deleted:
  - `downloadBlob` — was only called by `downloadFullArchive`; orphaned when the ZIP code was removed.
  - `valueOrEmpty` — pure markup helper, zero call sites anywhere in the file. Leftover from an earlier version.
  - `variantForPlan`, `variantForRisk` — one-line aliases for `variantForPdcStatus` and `variantForSupport` that nothing called.
  - `focusMeetingPreset` — a two-line wrapper around `openMeetingModal`, zero callers.
  - `renderCustomFieldsMarkup` — returned a markup string no render path consumed. Remnant of the custom-fields UI that v0.15.6 removed from the direct report workspace.
  - `bindWorkspaceCustomFieldControls` — bound handlers to `#addCustomFieldBtn` and `#detailCustomFieldsList`, neither of which are present in the current DOM. Same remnant as above. This function was already dead on arrival in v0.15.6.
- Each removal was verified by grepping for the function name as a whole word and for string-based references (e.g. `window['name']`) before deletion. After the deletions a re-scan reported zero remaining orphans.
- Known follow-up, deliberately not done in this hygiene release: the `detailCustomFieldsList` DOM element never exists, so inside `buildReportPayloadFromForm` the branch that calls `collectWorkspaceCustomFields` is unreachable, which leaves `collectWorkspaceCustomFields` effectively dead. Simplifying that would mean editing live control flow on the main save path, which is a behavioral refactor rather than scanner hygiene. Flagged for a future patch.

3. innerHTML reductions
- Rewritten to `createElement` and `textContent`:
  - `populateSelect` clear (`innerHTML = ''` -> `replaceChildren()`)
  - quarterly export reminder mount clear (`innerHTML = ''` -> `replaceChildren()`)
  - detail drawer close path clear (`innerHTML = ''` -> `replaceChildren()`)
  - team-health empty state (static text + one button)
  - reports table empty row (static text + one button inside a single table row)
  - data-health grid in Settings (twelve `<strong>`/`<span>` rows built in a loop; every value is now a `textContent` write, no escaping required)
- One further innerHTML site — the custom field row builder in `bindWorkspaceCustomFieldControls` — was rewritten mid-sweep, and then removed entirely in section 2 when the function was found to be unreachable. The initial rewrite is mentioned here only for completeness.
- Kept as `innerHTML` because the markup is structurally complex and all dynamic values already pass through `escapeHtml()`:
  - quarterly export reminder wrapper content
  - KPI stat tiles (dynamic classes, inline width styles, nested structure)
  - team-health-grid layout
  - reports table rows (multi-cell rows with badges, namecells, action buttons)
  - direct report workspace drawer shell
  - tile modal list body
- Each kept site now has a short in-file comment explaining why it is kept. A future hygiene pass will not need to re-derive the decision.

4. CSS whitespace compression
- The CSS block between `<style>` and `</style>` has been compressed by collapsing runs of whitespace, tightening spaces around `{ } ; : ,`, and dropping the redundant trailing `;` before `}`. The CSS now sits on roughly 270 lines instead of roughly 1720.
- No `/* */` comments were present in the CSS block, so nothing of that kind needed to be stripped.
- Rule count, selectors, declarations, `@media` queries, `@keyframes` animations, `calc()` expressions, and quoted font family names are all preserved unchanged.
- The JavaScript block is untouched. Only CSS whitespace was rewritten.

Compatibility
- No schema version change. Current schema version is still 2.
- Existing Team Pulse folders and JSON files are fully compatible.
- No migration is required for any field. No feature that writes into the durable document has been changed.
- Anyone relying on `teampulse_archive_YYYYMMDD.zip` as an output artifact should switch to the plain-format files in the Team Pulse folder plus `Export now` for CSVs and notes. The JSON, rolling backups, schema doc, and CSV/Markdown exports that the ZIP used to bundle are all still written individually.
- Total HTML file is noticeably shorter than v0.15.6 mainly because of CSS compression, and secondarily because of the removed ZIP code.
